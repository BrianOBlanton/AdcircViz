Welcome to the StormSurgeViz Home Page

StormSurgeViz is a MATLAB-based tool for visualization and analysis of CF-compliant model output funded by NOAA's Joint Hurricane Testbed (2013) Program (http://www.nhc.noaa.gov/jht/).

If you are a producer of storm surge model output, you may pariticpate in StormSurgeViz activities to make your model output visible in StormSurgeViz. By following an "onboarding" checklist of steps available here, you may participate in those activites.

Basic Implementation of StormSurgeViz

StormSurgeViz relies on making connections to an OPeNDAP server using the MATLAB toolbox nctoolbox. This toolbox (http://nctoolbox.github.com/nctoolbox) is used extensively to open netCDF data files, without accessing the data components until they are needed. Each graphic is generated using MATLAB functions and tools “on the fly”; the color map, color limits and color increment are specified by the user with default values appropriate to the scalar field being rendered. The default colormap is one similar to the NOAA colormap currently used for NHC surge maps:

All needed MATLAB code is included in the application’s distribution package. Other than having MATLAB, no other MATLAB or external dependencies exist.

QuickStart

Download StormSurgeViz

To download the most recent version of StormSurgeViz, go to the top of this page (http://renci-unc.github.io/StormSurgeViz/) and click on View in GitHub and then Clone in Desktop. You may need to install a GitHub client for your OS. You can also download a zip file of the code, but each time the code is updated, you will need to get a new zip file and delete the old downloaded directory/folder. Using the GitHub client will allow for easy updating of the code.

Add StormSurgeViz to the MATLAB PATH

The cloned repository will be put into a directory called StormSurgeViz, in a location specified by the user. Put this path into the MATLAB path, perhaps in the user’s or system startup.m file. For example:

SSVizHome='/home/myusername/GitHubRepos/StormSurgeViz';
          addpath(SSVizHome)

Run StormSurgeViz

Start MATLAB. To open the GUI window of the StormSurgeViz Tool, type StormSurgeViz in the Command Window to get a default view. 

StormSurgeViz

This should result in a large GUI that looks like the following:

The default catalog is RENCI's OPeNDAP server, and the most recent ADCIRC simulation entry is retrieved and displayed. Depending on when the user starts StormSurgeViz, the actual solution and details in the above figure will vary, but the format and layout of the GUI should like that shown.

Note: Only one instance of StormSurgeViz GUI is allowed to run at a time. Close the previous one before open another.

Change font size

If the fonts and GUI layout is cramped, change the fontsize of the GUI window by setting the FontOffset parameter to adjust, e.g., type 

close all;  StormSurgeViz('FontOffset',-2)

Change scalar field units

To change the units of the map (default “Meters” ) to Feet, use the Units parameter to adjust, e.g., type 

close all; StormSurgeViz('Units','Feet')

Point to a different Thredds Server

To view a different Thredds Server, use the ThreddsServer parameter to specify a different URL, as in 

close all; StormSurgeViz('ThreddsServer','http://coastalmodeldev.data.noaa.gov/thredds')

Point to a different catalog file

To specify a different catalog file, use the Catalog parameter to specify a different URL. For example, to view the catalog file with only the small, coarse-grid ec_95d test files on the default RENCI Thredds Server, type

close all; StormSurgeViz('CatalogName','catalog.tree.tc.ec95d')

Plotting vector fields

Vector plotting is enabled when the user checks on an available vector field (e.g., Wind Velocity) and sets a time in the Vector Snapshot List. To have both water level and wind vectors simultaneously, the user must also check the Water Level field. Parameters for tuning the vector drawing are accessible through the Vector Options panel, which is enabled by clicking on the VO button in the main MATLAB toolbar.
